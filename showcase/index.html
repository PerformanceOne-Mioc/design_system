What Makes a Dennis Mioc Deliverable Recognisable

01 Glass is structural, not decorative. Every surface is frosted. There are no flat white cards with drop shadows.
02 The palette is specific. Terra cotta is a sculptor's pigment, not a "warm CTA colour." The blobs behind the glass are chromatic, not just atmospheric.
03 The type is precise. Barlow SC KPI values read like instrument readouts. Atkinson body text is functionally unambiguous.
04 The hierarchy is architectural. The most important number is the biggest, boldest, most isolated element on the card. Everything else is context.
05 The layout is asymmetric. Nothing is centred for comfort. Left-alignment with considered spacing signals authorship.

Motion

Animate only transform and opacity. Expo-out for interactions, ease-in-out for ambience. Hover each card to feel the duration.

Easing Tokens

Default (expo-out)
cubic-bezier(.16,1,.3,1)
All interactive transitions — buttons, cards, panels. Natural deceleration.
Ease-in-out
ease-in-out
Blob drift, theme transitions, body background. Smooth pendulum motion.
Linear
linear
Spinner rotation only. Nothing else should ever use linear.
Blob Rules
  • →Only transform: translate + scale + rotate — never layout properties
  • →Four blobs, four different cycle durations (20s / 22s / 24s / 26s) — organic, non-synchronised
  • →animation-direction: alternate on all blobs — no restart flash
  • →prefers-reduced-motion stops all animations — no exceptions

Reference

Anti-Patterns

Strictly banned. These are the fingerprints of generic AI output — recognising them is part of the craft.

Typography
✕Inter, DM Sans, Plus Jakarta Sans, Outfit, Fraunces, Space Mono, Syne, IBM Plex Sans, Space Grotesk, Instrument Sans
✕Monospace as a "technical" style signal
✕All-caps on body copy or headings longer than a few words
✕Adjacent type sizes less than 1.25× apart — hierarchy must read
✕Fluid type in product UI — fixed rem scales only in dashboards
Colour
✕Pure #000 or #fff — always tint
✕Gray text on coloured backgrounds — use hue-tinted shade
✕Neon accents, cyan/purple "tech" palette, AI colour defaults
✕Gradient text — background-clip: text + gradient fill — absolute ban
✕More than one accent colour active in a view — one terra cotta only
✕Chroma above 0.25 anywhere in the palette
Components
✕border-left or border-right > 1px as colour accent stripe — this is the single most overused AI tell in dashboards
✕Identical card grids — icon + heading + text repeated 3–4 times
✕Modals when drawer, inline-expand, or navigation would serve
✕Decorative sparklines — only use charts if they convey a real trend
✕Empty states with "No items" and no next action
✕Nested cards
Layout
✕Three equal-column layouts
✕Centering everything — left-aligned asymmetric compositions instead
✕Glassmorphism used decoratively rather than as the structural surface
✕Same padding/gap repeated on every element — vary for hierarchy
✕Body text beyond ~65–75ch per line
Motion
✕Animating layout properties — width, height, padding, margin, top, left
✕Bounce or elastic easing — real objects decelerate smoothly
✕Canvas confetti, particle systems, cursor trails
✕Animations without prefers-reduced-motion fallback — no exceptions
The Drift Test
→Glass is only cosmetic, not structural — surfaces feel decorative
→Terra cotta appears more than once as a visual accent
→KPI value competing with decorative elements on the same card
→Layout is symmetric — everything centred "for comfort"
→If someone looks at this and immediately thinks "AI made it" — that's the failure
